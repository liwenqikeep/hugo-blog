## 前言

Java 集合框架（Java Collections Framework）是 Java 开发中最常用、最核心的基础设施之一。从日常的业务代码到底层的框架实现，集合无处不在。

本系列将用 **10 篇文章**，从整体到局部、从原理到实战，系统地梳理 Java 集合框架的核心知识点。

这是系列的第一篇，让我们先从全局视角，鸟瞰整个集合框架的架构。

## 两大根接口：Collection vs Map

```
┌─────────────────────────────────────────────────────────────────┐
│                         Collection                               │
├─────────────────────────────────────────────────────────────────┤
│  • 代表一组对象（元素）                                            │
│  • 每个元素是独立的对象                                             │
│  • 主要方法：add(), remove(), contains(), iterator()              │
└─────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────┐
│                            Map                                   │
├─────────────────────────────────────────────────────────────────┤
│  • 代表一组键值对（Key-Value）                                     │
│  • Key 唯一，Value 可重复                                         │
│  • 主要方法：put(), get(), containsKey(), entrySet()              │
└─────────────────────────────────────────────────────────────────┘
```

> **简单记忆**：`Collection` 是元素的集合，`Map` 是键值对的映射。

## 三大接口：List、Set、Queue

### List - 有序、可重复

```
┌────────────────────────────────────────────────────────────────┐
│                          List                                   │
├────────────────────────────────────────────────────────────────┤
│  • 元素按插入顺序存储                                             │
│  • 可以通过索引访问元素                                            │
│  • 允许重复元素                                                   │
│  • 主要实现：ArrayList、LinkedList、Vector                        │
└────────────────────────────────────────────────────────────────┘
```

**常见实现对比**：

| 实现类 | 底层数据结构 | 线程安全 | 适用场景 |
|--------|-------------|---------|---------|
| ArrayList | 动态数组 | 否 | 随机访问、遍历多 |
| LinkedList | 双向链表 | 否 | 频繁增删、栈/队列 |
| Vector | 动态数组 | 是 | 需要线程安全时 |

### Set - 无序、去重

```
┌────────────────────────────────────────────────────────────────┐
│                           Set                                   │
├────────────────────────────────────────────────────────────────┤
│  • 元素不重复（依赖 hashCode + equals）                           │
│  • 不保证元素顺序（部分实现保证顺序）                               │
│  • 主要实现：HashSet、LinkedHashSet、TreeSet                      │
└────────────────────────────────────────────────────────────────┘
```

**常见实现对比**：

| 实现类 | 底层数据结构 | 是否有序 | 适用场景 |
|--------|-------------|---------|---------|
| HashSet | HashMap | 无序 | 最常用、去重场景 |
| LinkedHashSet | LinkedHashMap | 插入顺序 | 需要保持插入顺序 |
| TreeSet | TreeMap | 自然顺序/自定义 | 需要排序 |

### Queue - 队列、先进先出

```
┌────────────────────────────────────────────────────────────────┐
│                         Queue                                   │
├────────────────────────────────────────────────────────────────┤
│  • 先进先出（FIFO）                                              │
│  • 主要方法：offer(), poll(), peek()                            │
│  • 主要实现：LinkedList、PriorityQueue、ArrayDeque               │
└────────────────────────────────────────────────────────────────┘
```

> **Tip**：Deque（双端队列）继承自 Queue，可以从两端添加/删除元素。

## Map 接口体系

### 核心 Map 实现

| 实现类 | 底层数据结构 | 是否有序 | 线程安全 | 适用场景 |
|--------|-------------|---------|---------|---------|
| HashMap | 数组+链表/红黑树 | 无序 | 否 | 最常用 |
| LinkedHashMap | HashMap+双向链表 | 插入顺序 | 否 | 需要保持顺序 |
| TreeMap | 红黑树 | 自然顺序 | 否 | 需要排序 |
| Hashtable | 数组+链表 | 无序 | 是（古老） | 不推荐 |
| ConcurrentHashMap | CAS+Synchronized | 无序 | 是 | 并发场景 |


## 实战选型指南

```
场景                              推荐选择
─────────────────────────────────────────────────────────────
需要快速随机访问                    ArrayList
频繁在任意位置增删                  LinkedList
需要元素唯一                        HashSet
需要保持插入顺序                    LinkedHashSet
需要元素排序                        TreeSet
键值对存储、查找                    HashMap
需要保持键值对的插入顺序            LinkedHashMap
需要按键排序                        TreeMap
多线程并发读写                      ConcurrentHashMap
```

## 常用方法速查

### Collection 核心方法

```java
// 添加元素
boolean add(E e)           // 添加单个元素
boolean addAll(Collection c) // 添加整个集合

// 删除元素
boolean remove(Object o)   // 删除单个元素
void clear()                // 清空所有元素

// 查询元素
int size()                  // 集合大小
boolean isEmpty()           // 是否为空
boolean contains(Object o)  // 是否包含元素

// 遍历
Iterator<E> iterator()      // 获取迭代器
```

### Map 核心方法

```java
// 添加/修改
V put(K key, V value)        // 添加键值对
void putAll(Map m)           // 添加整个Map

// 删除
V remove(Object key)         // 删除指定键

// 查询
V get(Object key)            // 获取值
boolean containsKey(Object key)  // 是否包含键
boolean containsValue(Object value) // 是否包含值
int size()                   // 键值对数量

// 遍历
Set<K> keySet()              // 获取所有键
Collection<V> values()       // 获取所有值
Set<Map.Entry<K,V>> entrySet() // 获取所有键值对
```

## 总结

1. **两大根接口**：`Collection` 是元素集合，`Map` 是键值对映射
2. **三大子接口**：`List` 有序可重复，`Set` 无序去重，`Queue` 队列
3. **最常用实现**：ArrayList、HashSet、HashMap
4. **线程安全选择**：多线程用 `ConcurrentHashMap`，单线程避免用 `Hashtable`
